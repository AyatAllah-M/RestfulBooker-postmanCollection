# Restful Booker API Testing Project

## Project Objective

The objective of this project is to gain practical experience in API testing by validating RESTful endpoints using Postman. The focus is on verifying request and response data, HTTP status codes, and overall API behavior across different scenarios.

## Scope of Testing

The following API functionalities were tested:

Authentication (Create Token)

Create Booking

Get Booking Details

Update Booking (PUT)

Partial Update Booking (PATCH)

Delete Booking

## Tools & Technologies Used

Postman

RESTful APIs

HTTP Methods (GET, POST, PUT, PATCH, DELETE)

JSON

## Key Learnings

Creating and organizing Postman collections

Validating API responses and status codes

Working with authentication tokens

Restful Booker API
